# ASR-0014

**Priority:** 5200
**Status:** Active
**Created:** 2026-04-10
**Updated:** 2026-05-14

## Requirement

Each layer boundary must be crossed exclusively through its defined contract — no caller may access implementation details of the layer below it.

## Notes

Sourced from old ASR-018. Clarified on 2026-05-14 for persistence architecture: callers cross the persistence boundary only through `Database` public sub-facades. Components, workflows, and services must not import Tier 2 `*_aql.py` modules, manage `_id`/`_key` mechanics, or perform follow-up database choreography that belongs inside persistence.
