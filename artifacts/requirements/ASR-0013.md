# ASR-0013

**Priority:** 5100
**Status:** Active
**Created:** 2026-04-10
**Updated:** 2026-05-14

## Requirement

The persistence layer must expose a complete caller-facing intent API for each persistence domain. Callers must be able to request the operation they are performing without orchestrating collection-level writes, edge management, state initialization, or other database mechanics themselves.

## Notes

Sourced from old ASR-017. Clarified on 2026-05-14 to make completeness a public sub-facade contract (`db.library`, `db.ml`, `db.app`) rather than a requirement to expose storage-shaped collection primitives. Tier 2 AQL bindings own all multi-collection side effects needed to fulfill a Tier 3 intent method.
